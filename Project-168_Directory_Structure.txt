Notes..............
Rationale for the Merged Structure:
Logical Separation: The new raw/ and processed/ directories create a clear data pipeline, separating original source material from data transformed for application use. This is a standard and robust practice.

Consistency with Existing Data: Your existing data/core/lemegeton-goetia.csv fits perfectly into this new schema. It is effectively already a processed file (a curated list of Goetic demons). If you acquired the raw source text for the Ars Goetia in the future, it would belong in data/raw/lemegeton/ars_goetia/.

Scalability: This structure can easily accommodate other parts of the Lemegeton (e.g., Ars Notoria) and other grimoires (e.g., The Sacred Magic of Abramelin the Mage) by adding new folders under raw/ and processed/.

Clarity: The purpose of each data file is now clear from its location. The core/ directory remains for fundamental, system-critical data, while raw/ and processed/ handle more extensive textual and grimoire data ingestion.

This merged structure enhances organization without disrupting your existing work, making the repository more scalable and maintainable.
